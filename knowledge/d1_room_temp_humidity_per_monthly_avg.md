**2.1 Panel**

**2.1.1 ROOM TEMP & HUMIDITY PER MONTHLY AVG**

งานนี้ คือ ROOM TEMP & HUMIDITY PER MONTHLY AVG คือ Dashboard Panel ประเภท Time Series ที่ถูกออกแบบมาเพื่อวัดอุณหภูมิและความชื้นของแต่ละวัน และอุณหภูมิเฉลี่ยและความชื้นเฉลี่ยต่อเดือน บน Grafana ที่แสดง ค่าเฉลี่ยอุณหภูมิ (Room Temperature) และค่าเฉลี่ยความชื้นสัมพัทธ์ (Room Humidity) ของห้อง/พื้นที่ผลิตแบบฉีดพลาสติก (Injection Molding) โดยสรุปเป็นค่าเฉลี่ยรายวันและแสดงแนวโน้มต่อเนื่องเป็นรายเดือน ตลอดช่วงเวลาที่ผู้ใช้เลือก

กล่าวโดยสรุป Panel นี้ตอบคำถาม 3 ข้อหลัก:

1.  "ห้องผลิตมีอุณหภูมิ/ความชื้นแค่ไหนในแต่ละช่วงเวลา" — ผ่านเส้น room_temp (สีเขียว) และ room_humid (สีเหลือง)

2.  "ค่าที่วัดได้เทียบกับค่ากลาง (Baseline) เป็นอย่างไร" — ผ่านเส้นอ้างอิง avg_temp_line และ avg_humid_line

3.  "ในวันไหนที่เกิดชิ้นงาน NG บ้าง และเป็นของ Lot/Part ใด" — ผ่านจุด ng_temp_marker (สีแดง) และ ng_humid_marker (สีม่วง) ซึ่งเชื่อมกลับไปยังข้อมูล Lot จริงใน Table View

**2.1.1.1 วัตถุประสงค์**

Panel ROOM TEMP & HUMIDITY PER MONTHLY AVG ถูกสร้างขึ้นเพื่อ:

1.  เฝ้าระวังสภาพแวดล้อมการผลิต (Room Condition Monitoring) — อุณหภูมิ (Room Temp) และความชื้น (Room Humidity)

2.  สรุปแนวโน้มค่าเฉลี่ยรายเดือน (Monthly Average Trend) เพื่อให้ผู้บริหาร/หัวหน้าไลน์เห็นภาพรวมของสภาพห้องผลิตในระยะยาว ไม่ใช่แค่ค่า ณ จุดใดจุดหนึ่ง

3.  เชื่อมโยงข้อมูลสภาพแวดล้อมเข้ากับข้อมูล Lot การผลิตจริง (Part Name, Material No., Lot Material, Condition STD, QC Member) เพื่อให้สามารถสอบย้อนกลับ (Traceability) ได้ว่า Lot ใดผลิตภายใต้สภาพห้องแบบใด

**2.1.1.2 Business Rules**

| Rule | กฎ (Rule)                                                                                                                                                                                        | เหตุผล                                                                                           |
|------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------|
| 1    | ค่าที่แสดงบนกราฟเป็น ค่าเฉลี่ยรายวัน (Daily Average) ของ room_temp และ room_humid ไม่ใช่ค่า Raw ทุก Record                                                                                                   | ป้องกันไม่ให้ Lot ที่มีจำนวน Record มากผิดปกติในวันเดียวไปถ่วงน้ำหนักค่าเฉลี่ยเกินจริง                             |
| 2    | เส้น avg_temp_line / avg_humid_line (เส้นประ) คือ ค่าเฉลี่ยของค่าเฉลี่ยรายวัน (Average of Daily Averages) ตลอดช่วงเวลาที่เลือก ไม่ใช่ Average ของ Raw ทุกแถว                                                      | สอดคล้องกับหลักการเดียวกับ Rule #1 เพื่อให้เส้น Reference สะท้อนแนวโน้มจริง ไม่ถูกบิดเบือนจากความถี่ของการสุ่มตรวจ |
| 3    | จุด ng_temp_marker (สีแดง) / ng_humid_marker (สีม่วง) แสดงค่าเฉลี่ยอุณหภูมิ/ความชื้นห้อง ของ Lot การผลิตที่มีชิ้นงานถูกตัดสิน Judgement เป็น "ไม่ผ่าน (NG)" ในวันนั้น — ไม่เกี่ยวข้องกับ Condition STD หรือเกณฑ์มาตรฐานห้องใด ๆ ทั้งสิ้น | เพื่อแยกความผิดปกติ (Exception) ออกจากค่าปกติอย่างชัดเจนบนกราฟเดียวกัน โดยไม่ต้องเปิด Panel แยก              |
| 4    | คอลัมน์ Room_Temp_C และ Humidiy ในฐานข้อมูลเป็นชนิด VARCHAR(MAX) (ไม่ใช่ตัวเลขโดยตรง) และชื่อคอลัมน์ Humidiy สะกดผิดโดยตั้งใจตามฐานข้อมูลต้นทาง                                                                     | ต้องแปลงชนิดข้อมูล (TRY_CAST) ก่อนคำนวณเสมอ ฉะนั้นจะเกิด Error หรือค่าผิดพลาด                              |
| 5    | Panel ใช้ Filter ร่วมกับ Dashboard หลัก ได้แก่ Month, Year, Mold No., M/C No., Part No. (Multi-select ทั้งหมด)                                                                                           | ให้ผู้ใช้สามารถโฟกัสเฉพาะช่วงเวลา/เครื่องจักร/ชิ้นงานที่สนใจ โดยไม่ต้องเปิด Dashboard ใหม่                       |
| 6    | ข้อมูลใน Table View (room_temp, room_humid, Part Name, Material No., OI No., Sample Part No., Lot Material, Condition STD, QC Member) คือ Record ระดับ Lot ที่ใช้เป็นแหล่งข้อมูลดิบให้กราฟด้านบน               | ทำให้ทุกจุดบนกราฟสามารถสอบย้อน (Trace) กลับไปหา Lot ของ QC Member ที่รับผิดชอบได้                         |
| 7    | ค่า Condition STD เป็นเกณฑ์เฉพาะของแต่ละ Part/Material (ผูกกับ Lot Material) ไม่ใช่ค่าคงที่ตัวเดียวทั้งโรงงาน                                                                                                   | Material แต่ละชนิด จึงต้องกำหนดเกณฑ์แยกตามวัสดุ                                                        |

**หมายเหตุ (ตัวอย่าง / ยกตัวอย่าง): ค่า Threshold เช่น "24.0–25.2 °C" หรือ "40–48 %H" ที่อาจปรากฏในเอกสารประกอบ เป็นเพียง ตัวอย่างประกอบความเข้าใจ (Illustrative Example) เท่านั้น ค่าจริงต้องอ้างอิงจากคอลัมน์ Condition STD ของแต่ละ Part/Material ใน Excel ข้อมูลปี 2026 ที่ใช้งานจริง ไม่ใช่ค่าคงที่ตายตัว**

**2.1.1.3 วิธีการใช้**

1.เลือกช่วงเวลา ผ่าน Filter Month และ Year ที่มุมซ้ายบน

(รองรับ Multi-select เช่น เลือก**หลาย**เดือนพร้อมกัน)

2. **เลือก Material / Mold / Part** ผ่าน Filter Mold No., M/C No., Part No.

> เพื่อกรองเฉพาะกลุ่มที่ต้องการตรวจสอบ

3. ปรับช่วงเวลา**แบบละเอียด** ได้ที่ Time Range ด้านขวาบน (เช่น 2025-01-01 06:42:04 ถึง 2026-07-29 07:00:00) สำหรับดูช่วงสั้นกว่ารายเดือน

4.สลับมุมมอง ระหว่าง **กราฟเส้น (Time Series)** และ **ตารางข้อมูลดิบ (Table View)** ได้ที่สวิตช์ Table view มุมขวาบนของ Panel

- โหมดกราฟ → ดูแนวโน้มและจุดผิดปกติ

- โหมด Table → ดูรายละเอียด Lot/Part แต่ละแถวเพื่อสอบย้อนกลับ

**2.1.1.4 วิธีการอ่าน**

**Panel นี้เป็น Time Series จะมีการอ่านตามสัญลักษณ์ดังนี้:**

| สัญลักษณ์ | สี | ความหมาย | แกนอ้างอิง |
|---|---|---|---|
| เส้นทึบ room_temp | สีเขียว | ค่าเฉลี่ยอุณหภูมิห้องรายวัน (°C) | แกนซ้ายสุด (สเกล °C ช่วงแคบ เช่น 22.8–26.2) |
| เส้นทึบ room_humid | สีเหลือง | ค่าเฉลี่ยความชื้นห้องรายวัน (%H) | แกนซ้ายรอง (สเกลกว้าง เช่น 17.5–70) |
| เส้นประ avg_temp_line | สีน้ำเงิน | ค่าเฉลี่ยอุณหภูมิ Reference ตลอดช่วงเวลาที่เลือก | แกนซ้ายสุด (คู่กับ room_temp) |
| เส้นประ avg_humid_line | สีส้ม | ค่าเฉลี่ยความชื้น Reference ตลอดช่วงเวลาที่เลือก | แกนขวา (คู่กับ room_humid) |
| จุด ng_temp_marker | สีแดง | ค่าเฉลี่ยอุณหภูมิห้องของ Lot ที่ผล QC ตัดสินเป็น NG (Judgement [S] = 'X') ในวันนั้น — ไม่ใช่การเทียบเกณฑ์ห้อง | วางตามค่าจริงบนแกน room_temp |
| จุด ng_humid_marker | สีม่วง | ค่าเฉลี่ยความชื้นห้องของ Lot ที่ผล QC ตัดสินเป็น NG (Judgement [S] = 'X') ในวันนั้น — ไม่ใช่การเทียบเกณฑ์ห้อง | วางตามค่าจริงบนแกน room_humid |

**หลักการอ่านที่สำคัญ:**

**1) แนวโน้มปกติ: เมื่อเส้น room_temp/room_humid แกว่งใกล้เส้นประ (avg_line) แสดงว่าค่าห้อง ณ ช่วงนั้นใกล้เคียงค่าเฉลี่ยรวมของช่วงเวลาที่เลือก**

**2) จุดสีแดง/ม่วง (NG Marker) ไม่ได้แปลว่า "ห้องผิดปกติ" แต่แปลว่า "ในวันนั้นมีชิ้นงานจาก Lot นี้ถูกตัดสินเป็น NG โดย QC" — จุดจึงถูกวางไว้ที่ค่าอุณหภูมิ/ความชื้นซึ่งเป็นสภาพห้อง ณ ช่วงเวลาที่ Lot นั้นผลิต เพื่อให้ผู้ใช้สังเกตด้วยตาว่ามีรูปแบบ (Pattern) ความสัมพันธ์ระหว่างสภาพห้องกับการเกิด NG หรือไม่ (เช่น NG เกิดถี่ขึ้นเมื่อความชื้นสูง) — การตีความความสัมพันธ์นี้เป็นหน้าที่ของผู้ใช้ ไม่ใช่ระบบตัดสินให้อัตโนมัติ**

**3) กลุ่มจุดสีแดง/ม่วงที่หนาแน่นในช่วงเวลาใดเวลาหนึ่ง บ่งชี้ว่าช่วงนั้นมีจำนวนชิ้นงาน NG มาก ควรเปิด Table View เพื่อดูว่าเป็น Part/Lot ใด แล้วเทียบกับค่า room_temp/room_humid ในช่วงเดียวกันเพื่อวิเคราะห์ความสัมพันธ์ต่อ**

**2.1.1.5 หน้าที่ของ Panel**

**2.1.1.6 กลไกการทำงานแบบ Step-by-Step**

Panel นี้ประกอบด้วย 4 Query ย่อย ที่ทำงานคู่ขนานกัน แล้วนำผลมาซ้อนแสดงบนกราฟเดียวกัน โดยรายละเอียดเชิงลึกของแต่ละขั้นตอนเป็นดังนี้:

โดย ข้อมูลอย่าง Condition STD, Lot Material, Part Name, M/C No., QC Member, OI No., Sample Part No. และ Lot Material ที่แสดงประกอบในทุกขั้นตอน เป็นเพียง **ข้อมูลอ้างอิงที่แนบมาด้วย** เพื่อให้สอบย้อนกลับได้ว่าเป็น Lot/ชิ้นงานใด ไม่ได้ถูกนำไปใช้ตัดสินหรือคำนวณค่าใด ๆ ในกลไกนี้เลย

**Step 1 — รับค่า Filter จากผู้ใช้**

| **รายการ**           | **รายละเอียด**                                                                                               |
|----------------------|-------------------------------------------------------------------------------------------------------------|
| ตัวกรองที่ผู้ใช้เลือกได้     | เดือน, ปี, หมายเลขแม่พิมพ์, หมายเลขเครื่องจักร, หมายเลขชิ้นงาน (เลือกได้หลายค่าพร้อมกันในแต่ละตัวกรอง)                       |
| หลักการทำงานของตัวกรอง | ถ้าผู้ใช้เลือก "ทั้งหมด (All)" ระบบจะไม่กรองเงื่อนไขนั้นเลย แต่ถ้าเลือกค่าเฉพาะเจาะจง ระบบจะแสดงเฉพาะข้อมูลที่ตรงกับค่าที่เลือกเท่านั้น |
| ขอบเขตข้อมูลที่ตายตัว     | ไม่ว่าผู้ใช้จะเลือกตัวกรองใด ระบบจะจำกัดขอบเขตเฉพาะข้อมูลงาน Injection (งานฉีดพลาสติก) เสมอ ไม่ปนกับงานประเภทอื่น           |
| การจับคู่ค่าเดือน         | ตัวเลขเดือน (1–12) ที่บันทึกในระบบ จะถูกแปลงเป็นชื่อย่อเดือน (Jan–Dec) ก่อนนำไปเทียบกับเดือนที่ผู้ใช้เลือกบนหน้าจอ                 |

**Step 2 — ดึงข้อมูลดิบ**

| **รายการ**              | **รายละเอียด**                                                                                                                           |
|-------------------------|-----------------------------------------------------------------------------------------------------------------------------------------|
| แหล่งข้อมูล                | ฐานข้อมูลกลางของโรงงานที่เก็บ Record การผลิต/ตรวจสอบคุณภาพทั้งหมด (แหล่งข้อมูลเดียวกันสำหรับทุกส่วนของ Panel นี้)                                          |
| เงื่อนไขความถูกต้องของวันที่   | Record ที่ไม่มีวันที่ผลิตบันทึกไว้ หรือวันที่บันทึกไว้อยู่ในรูปแบบที่ระบบแปลความหมายไม่ได้ จะถูกตัดออกจากการคำนวณทั้งหมด                                             |
| เงื่อนไขความถูกต้องของค่าห้อง | Record ที่ไม่มีค่า Room Temp หรือ Humidity บันทึกไว้ หรือค่าที่บันทึกไว้อยู่ในรูปแบบที่แปลงเป็นตัวเลขไม่ได้ จะถูกตัดออกจากการคำนวณเช่นกัน เพื่อไม่ให้ค่าผิดปกติไปกระทบผลลัพธ์ |

**Step 3 — แปลงชนิดข้อมูล**

| **รายการ**     | **รายละเอียด**                                                                                                                                              |

| หัวข้อ | สายคำนวณที่ 1 (เส้นแนวโน้มหลักรายวัน) | สายคำนวณที่ 2 (เส้นค่าเฉลี่ยอ้างอิงรวม) | สายคำนวณที่ 3 (จุดแสดงความสัมพันธ์กับชิ้นงาน NG) |
|---|---|---|---|
| ระดับการจัดกลุ่มข้อมูล | จัดกลุ่มตามวันที่ (รวมทุก Record ในวันเดียวกันเข้าด้วยกัน) | ไม่จัดกลุ่มตามวัน พิจารณาข้อมูลทั้งชุดในคราวเดียว | จัดกลุ่มตาม "Lot การผลิต" และ "วันที่" ร่วมกัน (พิจารณาเป็นรายล็อตต่อวัน) |
| วิธีคำนวณค่าเฉลี่ย | หาค่าเฉลี่ย Room Temp / Humidity ของทุก Record ในแต่ละวัน โดยไม่แยกว่าเป็น Lot การผลิตใดบ้าง | ขั้นแรกหาค่าเฉลี่ยรายวันของทุก Record ก่อน (เหมือนสายคำนวณที่ 1) จากนั้นนำค่าเฉลี่ยรายวันเหล่านั้นทั้งหมดมาหาค่าเฉลี่ยอีกครั้ง จนได้เป็นค่าคงที่เพียงค่าเดียว ที่สะท้อนภาพรวมทั้งช่วงเวลาที่เลือก | หาค่าเฉลี่ย Room Temp / Humidity ของแต่ละ Lot ในแต่ละวัน แล้วตรวจสอบว่า Lot-วันนั้นมีชิ้นงานที่ผล QC ตัดสิน Judgement เป็น "ไม่ผ่าน (NG)" ปะปนอยู่หรือไม่ |
| การเกี่ยวข้องกับชิ้นงาน NG | ไม่เกี่ยวข้อง — ใช้ข้อมูลทุก Record โดยไม่สนใจผลตัดสิน NG | ค่าตัวเลขของเส้นไม่เกี่ยวข้องกับ NG แต่ระบบจะหาว่าช่วงเวลาที่มีชิ้นงาน NG เกิดขึ้นครั้งแรกและครั้งสุดท้ายอยู่ช่วงใด เพื่อนำมากำหนดเฉพาะตำแหน่งเริ่ม-จบของเส้นบนกราฟเท่านั้น (ไม่กระทบตัวเลขของเส้น) | เก็บไว้เฉพาะ Lot-วันที่มีชิ้นงาน NG เกิดขึ้นจริงเท่านั้น Lot-วันใดไม่มี NG จะถูกตัดออกทั้งหมด |
| ตำแหน่งเวลาที่แสดงบนกราฟ | ใช้เวลาที่พบ NG ครั้งแรกของวันนั้น ถ้าไม่มี NG ใช้เวลาเริ่มต้นของวัน | ใช้เวลาของ NG ครั้งแรกสุดและครั้งสุดท้ายสุดในข้อมูลทั้งชุด เป็นจุดเริ่ม-จุดจบของเส้น | ใช้หลักการเดียวกับสายที่ 1: เวลาที่พบ NG ล่าสุดของ Lot-วันนั้น ถ้าไม่มีใช้เวลาเริ่มต้นของวัน |
| จำนวนจุดที่แสดงผล | 1 จุดต่อวัน ทุกวันที่มีข้อมูล | เพียง 2 จุดตลอดทั้งกราฟ (เริ่มต้น-สิ้นสุด) ค่าเท่ากันทั้งสองจุด ดูเป็นเส้นตรงแนวนอน | 1 จุดต่อวัน เฉพาะวันที่มี Lot เกิด NG |
| ข้อมูลประกอบที่แนบมาด้วย | Part Name, Material No., OI No., Sample Part No., Lot Material, Condition STD, QC Member ของวันนั้น (1 ชุดค่าต่อวัน) | Part Name, Material No., Lot Material และข้อมูลอื่นที่เกี่ยวข้องรวมเป็นชุดเดียว | Part Name, Material No., Lot Material เฉพาะของ Lot ที่เกิด NG ในวันนั้น |
| แสดงผลบนกราฟเป็น | เส้นทึบ (อุณหภูมิ=เขียว, ความชื้น=เหลือง) | เส้นประลากตรงแนวนอน (อุณหภูมิ=น้ำเงิน, ความชื้น=ส้ม) | จุดกระจาย (อุณหภูมิ=แดง, ความชื้น=ม่วง) |

โดยค่าตัวเลขของเส้นอ้างอิงคำนวณจากข้อมูลทั้งหมดในช่วงที่เลือกดู แต่ตำแหน่งที่วางเส้นบนแกนเวลา (จุดเริ่ม-จุดจบ) ถูกกำหนดโดยช่วงเวลาที่มีชิ้นงาน NG เกิดขึ้นเท่านั้น — ทั้งสองส่วนนี้เป็นคนละเรื่องกัน อย่าตีความว่าเส้นนี้คำนวณจากเฉพาะข้อมูล NG

**Step 5 — รวมผลลัพธ์ทั้ง 3 สายคำนวณ**

**2.1.1.9 Flow การทำงานแบบข้อความ (แทน Diagram 1: ภาพรวม Flow)**

ลำดับการทำงานของ Panel นี้ แบ่งเป็น 6 Step หลัก โดย Step 4 แตกออกเป็น 3 สายคำนวณอิสระที่ทำงานขนานกัน แล้วมารวมกันอีกครั้งที่ Step 5:

1. **Step 1 — รับ Filter จากผู้ใช้**: Month / Year / Mold No. / M/C No. / Part No. (เลือก "All" = ไม่กรอง)
2. **Step 2 — ดึงข้อมูลดิบ**: จากฐานข้อมูลกลางของโรงงาน เฉพาะงาน Injection เท่านั้น ตัด Record ที่ไม่มีวันที่หรือค่าห้องออก
3. **Step 3 — แปลงชนิดข้อมูล**: แปลง Room Temp และ Humidity จากข้อความ (TRY_CAST) ให้เป็นตัวเลขที่คำนวณได้ ทำแยกกันในแต่ละสายคำนวณ
4. **Step 4 — แยกเป็น 3 สายคำนวณอิสระ**:
   - **สายที่ 1 (เส้นแนวโน้มหลัก)**: จัดกลุ่มตามวันที่ → หาค่าเฉลี่ย Room Temp/Humidity ของทุก Record ในวันนั้น → ถ้าวันนั้นมี NG ใช้เวลา NG แรกของวันเป็นตำแหน่ง ถ้าไม่มีใช้เที่ยงคืน → ผลลัพธ์: เส้นทึบสีเขียว/เหลือง รายวัน
   - **สายที่ 2 (เส้นค่าเฉลี่ยอ้างอิงรวม)**: หาค่าเฉลี่ยรายวันของทุก Record ก่อน (เหมือนสายที่ 1) → นำค่าเฉลี่ยรายวันทั้งหมดมาหาค่าเฉลี่ยอีกครั้งจนได้ค่าคงที่ 1 ค่า → หาตำแหน่งเวลาของ NG ครั้งแรกสุด/ครั้งสุดท้ายสุด เพื่อกำหนดจุดเริ่ม-จบของเส้นเท่านั้น → ผลลัพธ์: เส้นประสีน้ำเงิน/ส้ม ลากตรงแนวนอน
   - **สายที่ 3 (จุดแสดงความสัมพันธ์กับ NG)**: จัดกลุ่มตาม Lot การผลิต + วันที่ร่วมกัน → หาค่าเฉลี่ย Room Temp/Humidity ของแต่ละ Lot ในวันนั้น → ถ้า Lot นั้นไม่มีชิ้นงาน NG ตัด Lot-วันนี้ทิ้ง ถ้ามี NG เก็บค่าเฉลี่ยไว้แสดงผล → ผลลัพธ์: จุด Scatter สีแดง/ม่วง เฉพาะช่วงที่เกิด NG
5. **Step 5 — รวมผลลัพธ์**: นำผลจากทั้ง 3 สายคำนวณมาซ้อนแสดงบนแกนเวลาเดียวกัน แต่ละสายอิสระต่อกันโดยสิ้นเชิง ไม่มีการส่งค่าระหว่างกัน
6. **Step 6 — แสดงผลตามโหมดที่ผู้ใช้เลือก**: โหมด Time Series → กราฟเส้น + เส้นอ้างอิง + จุดแสดงความสัมพันธ์กับ NG / โหมด Table View → ตารางรายละเอียด Lot (Part Name, Material No., Lot Material, Condition STD, QC Member ฯลฯ) สำหรับสอบย้อนกลับ

**2.2 ความสำคัญ**

1. **ช่วยตั้งข้อสังเกตเชิง Root Cause ของ NG ชิ้นงาน** — การเห็นสภาพห้อง ณ ช่วงเวลาที่เกิดชิ้นงาน NG (ผ่านจุด ng_temp_marker/ng_humid_marker) ช่วยให้ QC_Member ตั้งสมมติฐานได้ว่าสภาพแวดล้อมของห้องปฏิบัติงานใน Production Line อาจเป็นหนึ่งในปัจจัยที่เกี่ยวข้องหรือไม่ ทั้งนี้ Panel ไม่ได้สรุปสาเหตุให้อัตโนมัติ เป็นเพียงข้อมูลตั้งต้นสำหรับการวิเคราะห์ต่อ
2. **สนับสนุนการสังเกตแนวโน้มสภาพห้องเทียบกับค่าเฉลี่ยรวม** — เส้น avg_temp_line/avg_humid_line ซึ่งแสดงผลตลอดทั้งช่วงเวลาที่เลือก ช่วยให้เห็นว่าช่วงเวลาใดสภาพห้องเบี่ยงเบนจากค่าเฉลี่ยรวมมากผิดปกติ แม้ Panel จะยังไม่มีเกณฑ์ Threshold ที่ตายตัว แต่ก็เป็นข้อมูลอ้างอิงให้พิจารณาอุณหภูมิและความชื้นที่เกิดขึ้นได้
3. **เป็นหลักฐานสอบย้อนกลับ** — Table View ที่มีข้อมูลพื้นฐานการผลิต (Condition STD, Lot Material, Part Name, Material No., OI No., Sample Part No., QC Member) ที่ผูกติดกับ room_temp, room_humid, avg_temp_line, avg_humid_line เป็นตัวช่วยให้ QC Member วิเคราะห์ได้ว่า Lot ที่เกิด NG ผลิตภายใต้สภาพแวดล้อมภายในห้องปฏิบัติงานแบบใด ณ เวลานั้น (Condition STD ในที่นี้เป็นข้อมูลประกอบของ Part/Material นั้น ไม่ใช่เกณฑ์ที่ใช้ตัดสิน NG ของค่าห้อง)
